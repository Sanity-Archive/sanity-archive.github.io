Guide: Interactive Shells

This guide explains what an interactive shell is and how you can get it running.

Posted Apr 20, 2025
By S4nity
2 min read
Guide: Interactive Shells
Guide: Interactive Shells

What is an interactive shell?

It is a type of remote shell where the attacker gains full control over the target’s CLI (Command Line Interface), making the shell behave like a local terminal.
A good comparison is SSH vs. a regular reverse shell: in SSH, you can cancel commands by pressing [CTRL] + [C]. Doing that in a reverse shell stops netcat (or your listener), which closes the connection.

Why is it important?

Interactive shells allow interactions that would not be possible with a regular remote shell.
Some functionality of an interactive shell:

  • stopping commands via [CTRL] + [C]
  • using text editors like VIM or Nano
  • tab completion when navigating thru the system
  • using the arrow keys when editing commands
  • and more

How can I set it up?

Upgrading your shell

Depending on the shell you used you might need to upgrade your terminal first. There are multiple options for this but the most common is to use Python for it. To spawn a Python terminal you can run:

If Python isn’t installed on the target, you can try “script /dev/null -c bash” instead.

1

python3 -c 'import pty; pty.spawn("/bin/bash")'

Backgrounding the process

If you have never done this step before, don’t be shocked. It will drop you back to your local terminal.

Next you want to background your shell, by pressing [CTRL]+[Z].

Setup the terminal line settings

If you have never done this step before, don’t be shocked. This step will make your terminal look weird.

In this step we will tell our terminal to switch to raw mode:

  • Input is sent chracter by character, instead of waiting for [Enter]
  • It doesn’t handle commands like [CTRL] + [C] anymore

And also disable input echoing:

  • This prevents the console to print characters twice, or other strange outputs

fg is executed afterwards to jump back into the remote shell.

1

stty raw -echo; fg

Output your local TERM variable

Checking your local TERM variable is only needed once and can be skipped in future projects.
TERM is an environmental variable that specifies the kind of terminal you are using.

Open a new terminal, then display your local TERM variable via echo and then copy it’s output.

1

echo $TERM

Match the TERM of your target with your local TERM

Make sure to use whatever your TERM is instead of xterm-256color if it is something else.

Afterwards we overwrite the TERM variable inside the remote shell to match our local TERM:

1

export TERM=xterm-256color
