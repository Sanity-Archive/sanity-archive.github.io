HackTheBox - Validation

Validation is an easy Linux machine, where we have to exploit a SQL-injection vulnerability to get a foothold. Enumerating the system reveals, that only root can login to the machine. So we check for password re-use with the credentials of the database to gain access to the root account.

|_http-title: 502 Bad Gateway

We can see that SSH and two HTTP ports are open. SSH is only on very rare occasions a good target to start with and HTTP port 8080 responded with 502 Bad Gateway, which leaves us with attacking port 80.

Entering the website directs us to a signup page.

If we enter a name and click on “Join Now”, we are directed to a page that tells us who is also registered for the country we selected.

There are no fields for user input on this page, so we go back to the sign up page. If we enter a new name and click on “Join Now” it shows both, our new and old player name, which means that it might have a database in the backend. So we go back, enter a new name, click on “Join Now” and intercept the connection with BurpSuite.

Let’s think about the information we have, before we continue.

  • So, we have a user input field and a dropdown menu. It is very likely that the user input field is protected against injections. The dropdown menu on the other hand might be considered “safe” data, because you can only select an item from the list.

  • It is rather likely that there is a database running in the backend.

  • We saw in the account.php page a player name and a country, but if it is a database it is very likely that every entry has an ID, so it should look like this:
    ID:playername:country

So, let’s attack the dropdown menu with SQL-injection via union selection and three fields (because of ID,playername,country). A simple method is utilizing the @@version function, which will display the version and type of database.

It worked! Now let’s try to get a webshell running by injecting a php shell and trying to store it as a web page.

Now we need to navigate to “http://IP/shell.php?0=id”.

It worked! Let’s use the webshell to get a reverse shell going. First we need to start a listener:

1
2

nc -lvnp 8844
<...>

Then we need to intercept the connection with BurpSuite and inject a reverse shell. I used a PHP shell in this case, because we already know it is installed on the system.

The first screenshot shows the shell “as is”. The second picture is URL-encoded, which you can do by highlighting the text and the pressing [CTRL] + [U].

Afterwards I ran the command “/bin/bash -i” to be able to see in which directory we are currently in. Then I grabbed the user flag in the /home directory.

Escalating our privileges

First I checked /etc/passwd to see what other users accounts are on the machine.

1
2

cat /etc/passwd | grep '/bash'
root:x:0:0:root:/root:/bin/bash

We can see that only root is able to login.
Then I checked the config.php in the home directory of www-data (/var/www/html), to see if we can get credentials from there. It actually stores clear text credentials! Let’s try if they work for root:

2
3
4

su -
Password: ***
# It worked!
cat root.txt
